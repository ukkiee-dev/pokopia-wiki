# 리서치: 교차 검증 (Triangulation)

이 스킬은 3개 수집 산출물을 통합 평가해서 **무엇이 얼마나 확실한지** 등급을 매기는 방법을 표준화한다. 수집과 판단을 분리함으로써 자기 발견 편향을 제거.

## 왜 Triangulation인가

하나의 소스만 믿으면 그 소스의 편향을 그대로 상속한다. 셋이 독립적으로 모인 증거를 교차하면:
- **다중 지지** → 신뢰 강화
- **상충** → 숨은 편향 또는 조건 차이 발견
- **공백** → "아직 알려지지 않음" 명시

이 프로세스를 구조화하지 않으면 종합 보고서가 "인상 기반" 문서가 된다.

## Triangulation 프레임워크

### Step 1: Claim 추출

3개 파일에서 **사실 주장(factual claim)**과 **의견(opinion)**을 분리:

```
사실 주장: "X는 Y이다", "A는 B로 측정된다"
의견: "X는 좋다/나쁘다", "A가 B보다 낫다"
```

- 사실 주장 → Claim 매트릭스
- 의견 → 커뮤니티 여론 분포로 별도 정리

### Step 2: 소스 매핑

각 claim에 3가지 소스 유형에서 지지·반박 증거:

```
Claim C1: "{문장}"
- [Web]      source_name (1차, neutral) — URL  → 지지
- [Academic] Smith 2024, Nature (RCT, n=1245) → 지지
- [Community] 해당 사항 없음 (커뮤니티는 사실 증명 아님)
```

### Step 3: 상태 판정

| 상태 | 기준 |
|------|------|
| **다중 지지** | 2~3개 소스 유형에서 지지, 특히 1차 + peer-reviewed 포함 |
| **단일 지지** | 1개 소스 유형만 지지 |
| **상충** | 소스 유형 간 서로 다른 결론 |
| **공백** | 어느 소스에도 명확한 답 없음 |

### Step 4: 신뢰도 등급

```
A  — 다중 지지, 1차 증거, 방법론 공개, 최신 (6개월~2년)
B  — 2개 소스 유형 지지 또는 1차 단일
C  — 2차 소스만 or 학술 preprint 단일 or 커뮤니티 다수만
D  — 단일 저품질, 이해관계 있는 출처만
—  — 근거 불충분 (공백)
```

### Step 5: 편향 진단

체크리스트:
- [ ] 시차: 데이터가 현재와 몇 개월 간격?
- [ ] 지역: 영어권 중심인지, 한국 자료 있는지
- [ ] 이해관계: 지지 소스가 기업/기관 발표만인지
- [ ] 샘플링: 커뮤니티가 특정 플랫폼 편향인지
- [ ] 언어: 한 언어권 자료만 있는지
- [ ] 생존 편향: 성공 사례만 보였는지

### Step 6: 상충 원인 추정

상충 시 가능 원인을 나열만 함 (결정하지 않음):

| 원인 | 설명 |
|------|------|
| 시점 차이 | 과거 연구 vs 최근 |
| 조건 차이 | 실험실 vs 실사용 |
| 정의 차이 | X의 범위를 다르게 해석 |
| 이해관계 | 동기 있는 쪽의 주장 편향 |
| 측정 차이 | 단위·방법론 |
| 지역 차이 | 문화·제도 차이 |
| 오류·허위 | 한쪽이 잘못 |

writer가 맥락 포함해서 기술하도록 넘김.

## 신뢰도 등급 판정 세부

### A 등급 (매우 강함)

모두 충족:
- 1차 소스 또는 peer-reviewed 출처 최소 1개
- 독립 2개 이상 소스 유형 지지
- 최신 (주제 시효에 따라 1~3년 이내)
- 이해관계 편향 없음 또는 명시

### B 등급 (강함)

- 2개 소스 유형 지지 또는 1차 단일이어도 방법론·데이터 공개
- 최신 or 주제가 시효 덜 타는 경우

### C 등급 (약함)

- 2차 소스 체인 (뉴스→뉴스)
- 학술 preprint 단독
- 커뮤니티 다수 의견만 (사실 증명으로 불충분)

### D 등급 (매우 약함)

- 단일 저품질
- 이해관계 있는 출처만
- 저자/출처 미상

### — (공백)

- 추적 가능한 증거 없음
- 가설·추측

## 의견과 사실 분리 원칙

**사실(Claim):** Triangulation 대상
- "X는 Y이다"
- "A는 B% 효과 있다"

**의견(Opinion):** 여론 분포로만 별도 정리
- "X는 쓸만하다"
- "A가 B보다 낫다"
- "이 정책은 실패다"

의견은 Claim 매트릭스에 들어가지 않음. "여론 분포 섹션"에서 "커뮤니티에서 N%가 X를 쓸만하다고 평가"처럼 기술.

## 수집 산출물 간 소통 (간접)

수집 에이전트들은 서로 직접 소통하지 않음. 이 스킬이:
- 3개 파일을 읽고 통합
- 공백 중 결정적이면 해당 에이전트에 **재수집 요청** (1회)
- 오케스트레이터 경유

재요청 예:
```
"web-investigator에게 요청: C3 관련 {구체적 소스 제안} 추가 탐색"
```

## 금지 사항

- 자기 직접 수집·검색 (역할 밖)
- 한 소스 유형 결론적 폐기 (커뮤니티 완전 무시 등)
- 상충 한쪽 편듦 (양쪽 다 보존)
- 공백을 "문제 없음"으로 오인
- 커뮤니티 여론을 사실 증명으로 승격
- 주제 전문성 없이 방법론 품질 단언
- 의견과 사실 혼용
